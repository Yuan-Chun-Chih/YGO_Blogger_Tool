# PyInstaller + 程式碼簽章 + 發佈流程清單

## 1. 本機建置前檢查

- 確認 `venv` 已建立並可啟動專案
- 確認 `requirements.txt` 已安裝完成
- 確認程式本體可正常啟動
- 確認 `python .\main.py` 可正常啟動

## 2. 安裝依賴

```powershell
.\venv\Scripts\python.exe -m pip install -r .\requirements.txt
```

## 3. 本機驗證

```powershell
.\venv\Scripts\python.exe .\main.py
```

建議手動驗證：

- 選擇自己的 `cards.cdb` 與 `pics/` 後，搜尋卡片是否正常
- 卡圖預覽是否正常
- `複製圖片與文字` 是否可貼到 Word
- `複製匯出 HTML` 是否可貼到 Blogger HTML 檢視
- 貼上 HTML 草稿後是否仍保留原樣式

## 4. PyInstaller 打包

```powershell
.\build.ps1
```

預期輸出：

- `dist\YGOPRO_Blogger_Tool.exe`

## 5. 打包後驗證

- 在目前電腦直接執行 `dist\YGOPRO_Blogger_Tool.exe`
- 在另一台乾淨 Windows 電腦測試啟動
- 測試沒有 Python 環境時是否仍可執行
- 重新驗證卡片搜尋、卡圖預覽、Word 貼上、Blogger HTML 匯出

## 6. 程式碼簽章

### 可選方案

1. Microsoft Store
- 最穩定避免 SmartScreen 警告
- 由 Microsoft 重新簽章與發佈

2. 一般程式碼簽章
- 購買 OV 或 EV Code Signing Certificate
- 使用固定發行者持續簽章每次 release

3. Azure Trusted Signing / Artifact Signing
- 微軟官方建議的非 Store 簽章方案之一
- 適合自動化發佈流程

### 本機簽章範例

```powershell
signtool sign /fd SHA256 /tr http://timestamp.digicert.com /td SHA256 /a .\dist\YGOPRO_Blogger_Tool.exe
```

說明：

- 需要先安裝 Windows SDK 的 `signtool`
- 需要有效的程式碼簽章憑證
- 即使已簽章，新版本仍可能因 SmartScreen reputation 不足而出現警告

## 7. GitHub 發佈流程

### 初始化與提交

```powershell
git init
git branch -M main
git add .
git commit -m "Initial commit"
```

### 綁定遠端

```powershell
git remote add origin https://github.com/Yuan-Chun-Chih/YGO_Blogger_Tool.git
```

### 推送

```powershell
git push -u origin main
```

## 8. 發佈建議

- 原始碼發佈到 GitHub
- `exe` 建議放在 GitHub Releases
- Release 內容建議附上：
  - 版本號
  - 更新內容
  - 已知限制
  - SmartScreen / 簽章說明

## 9. SmartScreen 現實限制

- 無簽章：高機率出現警告
- 有簽章：仍可能因新檔案 hash 沒有 reputation 出現警告
- Microsoft Store：最接近可穩定避免警告的方案
