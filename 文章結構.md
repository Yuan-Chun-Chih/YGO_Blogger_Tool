# 文章結構草案

這份文件的目的，是先把 Blogger 文章的固定結構拆成明確區塊，方便後續把它做成程式欄位與自動產生流程。

目前依據：

- `HTML_template/html1`
- `HTML_template/html2`

注意：

- 目前模板原始檔有明顯編碼問題，所以這份文件先只整理「結構」與「欄位意圖」，不直接整理原文文字內容。
- 後續若要做成 MVP 表單，程式會依這份文件決定哪些區塊做成固定欄位，哪些區塊保留自由編輯。

---

## 1. 整體文章區塊順序

建議先統一成以下順序：

1. 頂部主視覺 Banner
2. `<!--more-->` 摘要分隔
3. 文章主標題
4. 文章副標題或短標語
5. 前言 / 本次主題簡介
6. 本次介紹卡片清單區
7. 卡片逐張介紹區
8. 主題分析 / 戰術分析區
9. 延伸補充區
10. 結語
11. 文末導覽 / 固定連結表格 / 注意事項

---

## 2. 區塊定義

### A. 頂部主視覺 Banner

用途：

- 放文章封面圖
- 建立文章第一眼辨識度

建議欄位：

- `hero_image_url`
- `hero_image_alt`

備註：

- 通常是整篇文章最上方的大圖
- 如果固定由使用者手動上傳 Blogger 圖片後貼網址，程式可保留一個欄位讓使用者填

### B. `<!--more-->` 摘要分隔

用途：

- 控制 Blogger 列表頁摘要截斷位置

建議欄位：

- 無

備註：

- 這是固定插入項
- 建議程式永遠自動輸出，不讓使用者手動刪

### C. 文章主標題

用途：

- 顯示本篇主題名稱

建議欄位：

- `article_title`

常見內容：

- 主題牌組名
- 新卡系列名
- 單卡評測主題名

### D. 文章副標題或短標語

用途：

- 補充主題定位
- 放一小句摘要、標語、吐槽或系列副題

建議欄位：

- `article_subtitle`

### E. 前言 / 本次主題簡介

用途：

- 交代這篇在介紹什麼
- 為什麼值得看
- 簡單講環境定位、玩法、更新重點

建議欄位：

- `intro_paragraphs`

型態：

- 多段純文字

### F. 本次介紹卡片清單區

用途：

- 在正式逐張展開前，先列出本篇重點卡片或目錄

建議欄位：

- `section_cards_title`
- `section_cards_intro`
- `featured_card_ids`

備註：

- 這區可以只有標題加一小段引言
- 卡片清單本身可由選入的卡片自動生成

### G. 卡片逐張介紹區

用途：

- 文章核心內容
- 每張卡片都包含圖片、卡名、效果、作者評論

建議欄位：

- `cards[]`

每張卡片建議結構：

- `card_id`
- `card_name`
- `image_url`
- `use_local_image_preview`
- `card_effect_text`
- `card_comment`
- `card_tagline`
- `card_order`

每張卡片實際顯示建議順序：

1. 卡圖
2. 卡名
3. 效果原文
4. 作者評論
5. 額外補充句

備註：

- `card_name`、`card_effect_text` 預設由 `cards.cdb` 讀取
- `image_url` 若未提供，匯出時先保留 placeholder
- `card_comment` 應該是表單主要輸入區
- `card_id` 只作為內部查圖、查資料用途，不出現在最終文章

### H. 主題分析 / 戰術分析區

用途：

- 不是逐張講卡，而是整體談這個牌組或主題的運作方式

建議欄位：

- `analysis_title`
- `analysis_paragraphs`

常見內容：

- 展開思路
- 優勢與弱點
- 對環境的影響
- 與舊版本差異

### I. 延伸補充區

用途：

- 放主線之外但仍重要的補充內容

建議欄位：

- `extra_sections[]`

每個補充段落建議結構：

- `heading`
- `body`
- `image_url`

常見主題：

- 額外搭配卡
- 相容引擎
- 注意的規則點
- 實戰提醒

### J. 結語

用途：

- 收斂觀點
- 簡短總評
- 提供讀者最後結論

建議欄位：

- `outro_title`
- `outro_paragraphs`

### K. 文末導覽 / 固定連結表格 / 注意事項

用途：

- 放固定頁面、Facebook、索引頁、其他導覽連結
- 放 AD Block 提示等全站共通文案

建議欄位：

- `footer_links_html`
- `footer_notice_html`

備註：

- 這區通常應視為固定模板的一部分
- 不建議每篇文章都重新編輯

## 3. 哪些欄位應該做成「固定模板」

建議固定，不讓使用者每篇都重打：

- `<!--more-->`
- 文末導覽表格
- 文末注意事項
- 固定的字型、顏色、對齊方式
- 卡片區塊的 HTML 外框

---

## 4. 哪些欄位應該做成「使用者輸入」

建議做成表單欄位：

- 文章主標題
- 文章副標題
- 前言
- 本次介紹的卡片清單
- 每張卡片的評論
- 主題分析
- 延伸補充
- 結語

其中卡片區塊的最低必填內容是：

- 卡名
- 效果文
- 作者評論

---

## 5. 哪些資料應該由程式自動帶入

來自 `cards.cdb`：

- 卡名
- 效果文
- ATK
- DEF
- Level

來自 `pics/`：

- 本機預覽圖片

由使用者另外提供：

- Blogger 可公開使用的圖片網址
- 主觀評論
- 分析段落
- 延伸閱讀連結

---

## 6. MVP 第一版建議只做的範圍

先做：

1. 載入固定模板
2. 編輯文章主標題 / 副標題 / 前言
3. 插入多張卡片介紹區塊
4. 編輯每張卡片評論
5. 編輯結語
6. 匯出 Blogger 可貼上的 HTML

先不做：

- 自動上傳圖片到 Blogger
- 自動抓相關文章
- 多模板切換
- 複雜拖拉排序 UI

請你確認以下幾點，之後我才能把這份結構直接對應到程式 UI：

   圖片 → 卡名 → 效果文 → 作者評論

  - 效果文
  - 作者評論
- 卡號不需要出現在最終文章

  - 前言
  - 卡片逐張介紹
  - 結語
